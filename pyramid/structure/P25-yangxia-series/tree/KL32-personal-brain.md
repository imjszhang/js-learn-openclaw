## SCQA

**S (情境)**
你有一个微信收藏夹。一个 flomo。一个 Notion 或者 Obsidian。
每个工具都花了不少时间打理。
你收藏了上千篇文章，记了几千条笔记，建了几十个标签。

**C (冲突)**
但当你需要写一篇文章、回答一个问题、或者只是想知道"我之前是不是写过关于 X 的内容"——
你打开第一个工具，搜不到满意的。
打开第二个工具，还是找不到。
第三个工具？上次打开是什么时候都不记得了。

收藏是收藏。笔记是笔记。知识库是知识库。
三个孤岛，三个搜索框，永远找不到真正想要的东西。

不是信息不够多，是信息太散了。

**Q (问题)**
有没有一个地方，能把你所有的信息聚合起来，然后替你思考？

**A (答案)**
有。我在 OpenClaw 里养了一个第二大脑。
不是 Notion，不是 Obsidian，不是一个需要我维护的仓库。
而是一个能主动告诉我"你之前写过关于 X 的内容，而且跟 Y 有关联"的私人大脑。

龙虾 + collector（微信文章）+ flomo（思考碎片）+ prism（结构化知识）= 私人智库。

## Key Line 树

### KL01 - 你的收藏、笔记、知识库，为什么永远是散的？

**功能：痛点切入**

**设计**：
打开你的微信收藏夹——900 篇。
打开你的 flomo——3000 条。
打开你的知识库——也许 200 个 atoms，60 个 groups。

你问自己一个简单问题：我关于"知识管理"都写过什么？

微信收藏夹搜一遍。flomo 搜一遍。知识库搜一遍。
三套结果，互相不关联。你不知道哪条笔记跟哪篇文章有关联。你不知道你三个月前的某个想法，其实跟昨天收藏的文章是同一个问题。

信息时代，每个人都在建自己的信息库。
但绝大多数人的信息库，是一座孤岛。

**开头类型**：痛点共鸣 + 反问（新变体：直接数字冲击）

---

### KL02 - 三个孤岛，三个搜索框

**功能：展开冲突——传统工具的碎片化**

**支撑**：
收藏、笔记、知识库，为什么天然就是散的？

因为它们的设计初衷就不一样：

| 工具 | 设计目的 | 输入方式 | 输出方式 |
|------|----------|----------|----------|
| 微信收藏 | "等一下再看" | 一键收藏 | 搜索（关键词匹配） |
| flomo | 记录想法 | 自由写作 | 搜索 + 标签筛选 |
| Notion/Obsidian | 结构化存储 | 分类、打标签、建链接 | 搜索 + 浏览 |

**问题**：
- 收藏的文章是别人的想法，但收藏时你没有任何备注
- flomo 是你的想法，但你不会每次都关联到具体文章
- 知识库是你整理的结构，但整理速度远跟不上收藏速度

三个工具的输入、输出、数据结构完全不同。
人工把它们合在一起？不可能。

**认知冲突**：
不是你不努力整理。是工具本身就是为了不同场景设计的，它们就不打算互通。

### KL03 - 我在 OpenClaw 里养了一个第二大脑

**功能：工具引入——三合一架构**

**支撑**：
于是我做了一件事：把三个数据源接到龙虾里。

```
微信文章 ──collector──┐
flomo 笔记 ──flomo MCP──┤── 龙虾（AI 整合）── 你的私人智库
知识库 ───prism──────┘
```

**三个入口，一个大脑**：

| 数据源 | 工具 | 角色 | 类比 |
|--------|------|------|------|
| 微信文章 | collector | 读过的书 | 记忆仓库 |
| flomo 笔记 | js-knowledge-flomo | 写下的想法 | 思考碎片 |
| 知识库 | prism | 整理过的经验 | 结构化知识 |

龙虾做的不是"搜索"——是把三套数据结构拆开、重新理解、再拼在一起。

微信文章被拆解成 atoms（信息单元）。
flomo 笔记被 AI 分析出洞察和关联。
prism 知识库自动归组、生成合成观点。

然后——
你问一个问题，龙虾从三个数据源里找答案。
不是拼凑三个搜索结果，而是真正理解后给你一个整合的回答。

**龙虾从"帮你管理信息"进化到了"替你整合信息"。**

### KL04 - 真实场景：私人智库能干么

**功能：实证展示——3 个场景**

**场景 1：写一篇文章**

我说："写一篇关于 Agent 生态的文章。"

如果没有私人智库：
1. 打开微信收藏夹，搜"Agent"——30 篇
2. 打开 flomo，搜"Agent"——50 条笔记
3. 打开知识库，搜"Agent"——15 条记录
4. 自己看完，自己整理大纲，从零开始

有了私人智库：
1. 告诉龙虾
2. 它自动从三个数据源找相关素材
3. AI 整合后给出框架："你收藏的 X 文章 + flomo 里 Y 观点 + 知识库 Z 结构，建议这样组织……"
4. 在框架上修改

从 3 小时 → 1 小时。

**场景 2：回答一个问题**

我问自己："我之前是不是讨论过'AI 会不会取代人类'这个话题？"

龙虾回答：
- flomo 里找到 12 条相关笔记，时间跨度 6 个月
- 知识库发现你从"AI 是工具"到"AI 是同事"到"AI 是生态"的认知变化
- 微信收藏找到 3 篇你读过但没评论的相关文章
- 总结："你的观点经历了 3 次转变，最新立场是……"

**场景 3：发现一个你不知道的东西**

龙虾主动说：
"你三个月前在 flomo 里写了一段关于'知识系统的复利效应'，今天又收藏了一篇关于'信息生态自生长'的文章。这两件事其实在说同一个问题——你可能正在构建一个理论。"

这是我永远不会自己注意到的关联。

### KL05 - 没有 vs 有私人智库的一天

**功能：反差对比**

**没有私人智库的一天**：
- 早上看到好文章 → 收藏到微信 → 然后忘了
- 想到一个好想法 → 记到 flomo → 然后忘了
- 想写篇文章 → 到处找素材 → 找了半天 → 算了下次再写
- 一周后，收藏夹里 50 篇未读，flomo 里 20 条无标签笔记

**有私人智库的一天**：
- 早上看到好文章 → 丢给龙虾一个链接 → 它自动总结、拆解、入库
- 想到一个好想法 → 记到 flomo → AI 分析后主动告诉我"这个想法跟之前的 X 有关联"
- 想写篇文章 → 告诉龙虾主题 → 它从三个数据源找齐素材、给出框架
- 一周后，系统多了 5 篇已分析的文章、3 条新发现的关联、1 个可写的框架

**核心差异**：
没有智库 → 信息越积越多，人越来越焦虑。
有智库 → 信息越积越多，系统越来越聪明。

### KL06 - AI 时代人人有同样的信息，差距在于什么

**功能：认知升级 + 结尾**

**支撑**：
回顾一下龙虾的成长轨迹：

KL05，装了收集袋——能看、能记。
KL06，装了棱镜——能消化、能产出。
KL27，有了手——微信数据自己去拿。
KL30，系统自生长——不用你维护。
KL31，开始替你思考——笔记它帮你读。
KL32，**完整的人——把三个孤岛连成一个大脑。**

信息时代最大的幻觉是"我收藏了 = 我拥有了"。
收藏不是拥有。
收藏 + 笔记 + 知识库 + AI 整合 = 真正的拥有。

AI 时代，人人都有同样的信息。
你跟我能看到同样的文章，同样的知识。
**差距不在于你知道什么，在于你有一个替你整合信息的大脑。**

> 你不用再怕信息太多了。
> 因为你不需要自己整理了。
> 你的第二大脑会先整理，整理完了告诉你它发现了什么。
