## 核心论点

**龙虾写代码太烧钱？教它学会「外包」——一条命令省下 90% Token**

很多人养了几周龙虾，token 额度嗖嗖往下掉。复盘发现：日常聊天其实不贵，真正烧钱的是让龙虾写代码——每次读文件、改代码、调工具，都在吃 token。OpenClaw 从 3.24 版本起支持 ACP（Agent 外包协议），让龙虾把编程活外包给 Cursor，Cursor 走自己的额度，龙虾只负责调度。就像公司老板不亲自写代码，而是交给外包团队——活照样干，钱省一大半。

## 三层支撑结构

### 25.12.1 第一层：编程是龙虾最烧钱的活（聊天不贵，写代码贵）

**核心观点**：你以为龙虾贵在聊天？其实 90% 的 token 花在了写代码上

**素材要点**：
- 常见误区：很多人以为是"聊多了"导致 token 用完，于是"省着聊"——方向错了
- 真正的成本大头：龙虾写代码时要做三件事，每一件都烧 token
  - **读文件**：龙虾想改你的代码，得先把整个文件读进来，几千行代码 = 几千 token
  - **写代码**：生成新代码、补全函数、修 Bug，每一行都消耗 token
  - **调工具**：跑测试、装依赖、查结果，每次工具调用都走 Chat 模型的上下文
- 一次编程任务可能消耗的 token = 你聊一天的量
- 类比：日常聊天是发微信（几毛钱一条），编程是打国际长途（几块钱一分钟）——你以为手机贵是因为微信发多了，其实是国际长途打多了

**引用 atoms**：AC-01（编程消耗根因）, AC-17（结构性必要措施）

**关键金句方向**：
- "你以为龙虾贵在聊天？错了。聊天是毛毛雨，写代码才是吞金兽"
- "省 token 不是省着聊，是别让龙虾亲自写代码"

---

### 25.12.2 第二层：ACP 外包术——龙虾当老板，Cursor 当程序员

**核心观点**：OpenClaw 的 ACP 机制让龙虾把编程外包给 Cursor，自己只管调度，token 成本从龙虾侧转移到 Cursor 侧

**素材要点**：
- ACP 是什么？一句话：龙虾编程时不再自己动手，而是叫 Cursor 来干
  - 全称 Agent Client Protocol（不用记，知道是"外包协议"就行）
  - 从 OpenClaw 3.24 版本开始支持
- 类比理解：
  - **以前**：龙虾 = 全栈打工人。你说"帮我写个脚本"，龙虾亲自读文件、写代码、跑测试，全程消耗你的 OpenClaw token
  - **现在**：龙虾 = 项目经理。你说"帮我写个脚本"，龙虾说"好的"，然后把任务派给 Cursor（外包程序员），Cursor 用自己的额度干活，干完把结果交回来
- 为什么是 Cursor？
  - Cursor 有自己的计费体系（免费额度 + Pro 订阅），不走 OpenClaw 的 token
  - Cursor 本身就是专业的编程 AI，写代码比龙虾更专业
  - 龙虾的优势是记忆、调度、多渠道协作——编程从来不是它的强项
- 关键理解：这不是"降级"，是"分工"
  - 龙虾负责：理解你的意图、规划任务、验收结果、记住教训
  - Cursor 负责：读代码、改代码、跑命令
  - 各干各擅长的，总成本反而更低
- 数据流极简版：你发消息 → 龙虾理解意图 → 龙虾派活给 Cursor → Cursor 写代码 → 结果返回龙虾 → 龙虾汇报给你
- 不阻塞日常对话：外包任务有独立通道，你可以一边让 Cursor 写代码，一边继续和龙虾聊别的

**引用 atoms**：AC-02（ACP 外包机制）, AC-03（两条线）, AC-13（独立会话隔离）

**关键金句方向**：
- "好员工不是事事亲为——懂外包的龙虾才是好龙虾"
- "龙虾当老板，Cursor 当程序员：各司其职，你省钱"
- "编程这件事，让专业的来。龙虾的强项是记忆和调度，不是写代码"

### 25.12.3 第三层：一条命令激活外包模式

**核心观点**：配置 ACP 外包不需要编程知识，一条命令自动搞定

**素材要点**：
- 前提条件：
  - 已安装 OpenClaw（3.24+）
  - 已安装 Cursor（免费版即可，Pro 更好）
  - 已安装 js-cursor-agent 插件
- 激活步骤（极简版）：
  1. 安装插件：`openclaw plugins install js-cursor-agent`
  2. 一条命令配好一切：`openclaw cursor setup`
  3. 重启网关：`openclaw gateway restart`
  4. 验证成功：在聊天里输入 `/acp doctor`，看到绿色就表示 OK
- `openclaw cursor setup` 自动干了什么？
  - 在配置文件里加入 ACP 外包开关（`acp.enabled: true`）
  - 指定外包对象为 Cursor（`acp.backend: "cursor"`）
  - 设置并发上限和超时参数
  - 支持 `--dry-run` 先预览不执行
- 激活后的日常使用：**完全无感**
  - 你照常和龙虾聊天、下指令
  - 龙虾自动判断哪些任务需要编程，自动外包给 Cursor
  - 你不需要手动切换，不需要学新命令
- 监控（可选）：
  - `/acp status`：看外包是否在线
  - `/acp sessions`：看当前有几个外包任务在跑
- 可选进阶：在插件配置里指定 Cursor 使用的模型（默认 composer-1.5，可切换为 claude-4.6-sonnet 等）

**引用 atoms**：AC-07（配置激活）, AC-08（插件配置）, AC-11（setup 命令）, AC-15（监控命令）, AC-16（模型列表）

**关键金句方向**：
- "一条命令，以后龙虾写代码再也不烧你的钱了"
- "配完之后你啥都不用管——龙虾自己知道什么时候该叫外包"
- "验证方式：输入 /acp doctor，绿色就对了"

## 叙事结构（文章大纲）

### 开场：Token 焦虑

- 养虾第 38 天，突然发现本月 token 额度快见底了
- 复盘消耗记录：80% 花在了"帮我写个 xxx 脚本""帮我改下 xxx 配置"上
- 不是聊太多了——是让龙虾写代码太贵了
- 引出问题："有没有办法让龙虾继续帮我编程，但不烧我的钱？"

### 论证：三层理解

1. **认知纠偏**：编程才是吞金兽，聊天是毛毛雨
2. **方案理解**：ACP = 龙虾学会了外包，Cursor 当程序员
3. **实操指南**：一条命令激活，日常完全无感

### 实战穿插：真实体验

- 激活 ACP 前：让龙虾帮忙写个收集器插件，一个下午烧了 X 万 token
- 激活 ACP 后：同样的编程任务，龙虾侧 token 消耗降到以前的 1/10
- 体验差异：速度没变慢，质量可能还更好（Cursor 本来就是写代码的专家）

### 收尾：好员工懂外包

- 回扣 KL09："把龙虾当员工"——好员工的标志不是事事亲为，是知道什么活该自己干、什么活该外包
- 龙虾的核心价值：记忆你的偏好、理解你的意图、协调多个工具——不是亲手写代码
- 插件开源：js-cursor-agent，一条命令装好
- "让龙虾回归它最擅长的事：当你的贴身管家，不是你的码农"
