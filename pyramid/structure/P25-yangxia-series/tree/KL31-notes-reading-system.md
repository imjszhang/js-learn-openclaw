### KL02 - flomo 开放了 MCP：一个让龙虾能用你笔记的信号

**功能：新闻引入 + 承上启下**

**支撑**：
flomo 六周年那天，发了一篇文章。
标题很克制：「让记录的意义，自然涌现」。

但里面藏了一个很关键的信息：

**flomo 全面开放了 MCP 接口。**

14 个工具。覆盖了笔记和标签的增删改查。
支持市面上所有支持 MCP 协议的服务——Claude、ChatGPT、Kimi、Cursor……

官方原文有一句：
> "和 AI 推演产品方案，聊到有价值的结论时，让 AI 提炼关键内容写入 flomo。
> 写文章？直接去 flomo 中查找相关素材。
> 标签很乱？让 AI 来自动构建一套符合你习惯的标签系统。"

flomo 官方自己在暗示一件事：
**你的笔记，不应该只躺在 flomo 里。**
**它应该能被 AI 读取、分析、调用。**

更有趣的是，他们在文章里提到了"小龙虾"：
> "年初小龙虾上线之后，我们曾尝试把自己的笔记压缩成它能使用的记忆，效果让我们惊艳。"

他们把 flomo 笔记压缩成 memory.md，替换了 OpenClaw 龙虾的默认记忆。
效果：AI 不再是"健忘的聪明朋友"，而是"知道了你是谁，你经历过什么，你在意什么"。

但问题来了。

flomo 给了接口。给了 MCP。给了 14 个工具。
然后呢？

**谁帮你的龙虾装上这个接口？**

### KL03 - js-knowledge-flomo：让笔记自己读自己

**功能：工具引入——因为 flomo 开放了 MCP，所以我开发了它**

**支撑**：
所以，我做了 js-knowledge-flomo 这个项目。

flomo 开放了 MCP，但 MCP 只是"接口"，不是"能力"。
接口是路，能力是车。
路修好了，你得有车才能开。

js-knowledge-flomo 做的事情：
- 用 flomo 官方 MCP 协议做 OAuth 授权，安全读写你的笔记
- 在笔记基础上叠加 AI 分析能力——洞察、关联、演变、写作
- 所有分析走你自己的 LLM API，数据不出你的环境
- **纯本地优先**：OAuth 凭据与本地缓存都在本机，无上报、无埋点

flomo 负责"让意义浮现"。
龙虾负责"替你发现意义"。

**三种用法**：
| 方式 | 入口 | 适用场景 |
|------|------|----------|
| OpenClaw 插件 | `openclaw flomo <命令>` | 在龙虾里直接对话式使用 |
| 独立 CLI | `node cli/cli.js <命令>` | 终端命令行，独立运行 |
| MCP Server | Cursor / Claude Desktop | 在 IDE 里当 MCP 工具调用 |
| Web UI | `npm run dev` | 本地网页搜索/统计 |

### KL04 - AI 替你读笔记：7 个维度

**功能：核心能力展示**

| AI 工具 | 解决的问题 | 你问 | 龙虾答 |
|---------|-----------|------|--------|
| **洞察发现** | 我记了这么多，到底在关心什么？ | "最近 30 天的笔记里我关心什么？" | 反复主题 + 情绪变化 + 隐藏关联 |
| **关联发现** | 这些笔记之间有什么关系？ | "帮我找找笔记间的隐藏联系" | 跨标签/跨时间的 3 组关联 + 推测大问题 |
| **观点演变** | 我对这个问题的看法是怎么变的？ | "追踪我对 Agent 的观点变化" | 时间线上的关键转折点 + 最新立场 |
| **写作大纲** | 我想写个东西怎么组织？ | "帮我用笔记素材写一篇关于 X 的大纲" | 结构化大纲（基于真实素材） |
| **素材搜集** | 写 X 主题有哪些素材？ | "搜集关于知识管理的笔记" | 直接匹配 + 关联推荐 |
| **标签建议** | 笔记没分类懒得打标签 | "帮我给无标签笔记推荐标签" | 基于现有标签树的 AI 建议 |
| **标签审计** | 标签越打越乱怎么优化 | "分析我的标签体系有什么问题" | 重复/命名/层级问题 + 优化方案 |

**核心突破**：
之前你面对 flomo 的方式是"搜索"——你知道自己要找什么。
现在变成"发现"——系统告诉你你没想到的东西。

### KL05 - 真实案例：从我的笔记里发现了什么

**功能：实证展示**

**设计**：
拿我自己的 flomo 做个实验。

**实验 1：洞察发现**
- 输入：最近 30 天，50 条笔记
- AI 返回 3 个反复主题 + 2 个情绪变化 + 1 个隐藏关联
- 总结："你最近真正在关心的是 X"

**实验 2：观点演变**
- 输入：追踪"Agent"话题
- AI 梳理出 4 个转折点
- 从"工具"到"同事"到"基础设施"的认知升级轨迹

**实验 3：关联发现**
- 输入：全部笔记（限定 50 条）
- AI 发现跨标签的隐藏关联
- 推测"你可能在思考的大问题"

### KL06 - 在 OpenClaw 里怎么用

**功能：场景演示**

**支撑**：
最舒服的用法：直接在飞书/微信里跟龙虾对话。

```
你：帮我看看最近 30 天的 flomo 笔记，我到底在关心什么？
龙虾：（调用 flomo_generate_insights）
龙虾：根据你最近的 50 条笔记，我发现你在反复思考这三个问题……

你：帮我追一下我对"知识管理"这个观点的演变
龙虾：（调用 flomo_track_evolution）
龙虾：你的观点经历了 4 个阶段……

你：我要写一篇关于 Agent 生态的文章，帮我搜集素材
龙虾：（调用 flomo_collect_material + flomo_draft_outline）
龙虾：找到 15 条相关笔记，大纲如下……
```

也可以在 Cursor 里配 MCP Server：
- 写代码时随时调 flomo 笔记
- 让 AI 从你的笔记库里找灵感

或者独立 CLI：
- 终端里一条命令生成洞察
- 不依赖 OpenClaw，任何人装了就能用

### KL07 - 龙虾开始替你思考了

**功能：认知升级 + 结尾**

**支撑**：
回顾一下龙虾的成长轨迹。

KL05，装了收集袋——能看、能记。

从"你喂龙虾"到"龙虾自己吃"到"龙虾替你消化"到"龙虾替你思考"。

知识管理的终点不是整理，是生长。
生长的终点不是堆积，是洞察。
洞察的上限不是你读了多少，是你的系统能替你想到多少。

**结尾金句**：

> 因为你不需要读了。
