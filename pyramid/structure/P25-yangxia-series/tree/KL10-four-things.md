## 三层支撑结构

### 25.10.1 第 1 件：给龙虾记性（memory-core 插件，能记住）

**核心观点**：没有记忆的龙虾，每次对话都是第一次见面

**素材要点**：
- 第 1-3 天：配置官方 memory-core 插件
- 背景：每次开聊都像第一次见面，之前说过的偏好、背景全忘
- 核心动作：
  - 在插件列表启用 memory-core（内置插件，无需额外安装）
  - 配置记忆槽位，确认 memorySearch.enabled 开启
  - 正常聊天，memory-core 自动同步索引
- memory-core 的"保存"本质是索引同步，质量取决于输入源而非聊天量
- 验收标准：龙虾能记住用户偏好，回答带有"我知道你是谁"的语气

**引用来源**：
- OpenClaw 官方 memory-core 插件文档
- journal/2026-02-28/memory-core-research-and-implementation-log.md

**关键金句**：
- "没有记忆的龙虾，每次对话都是第一次见面"
- "它真的记住我了"
- "质量取决于你喂进去的东西，而不是聊了多少句"

### 25.10.2 第 2 件：给龙虾一双眼睛（js-eyes 浏览器插件，能看见）

**核心观点**：龙虾看不见，等于瞎养

**素材要点**：
- 第 4-10 天：装 js-eyes 浏览器插件
- 背景：现在装好龙虾默认就连上飞书和微信，能说话已是常识
- 核心问题：龙虾能说话，但看不见——公众号文章、知乎回答等登录墙后的内容全部访问不了
- 核心动作：
  - 安装 js-eyes 浏览器插件
  - 右键抓取登录墙后的网页内容
  - 内容直接送进龙虾的知识入口
- js-eyes 同时被 collector 用于登录型平台抓取（与第 3 件事联动）
- 验收标准：龙虾能"看到"并消化登录墙后的文章内容

**引用来源**：
- KL03-js-eyes-login.md（浏览器自动化）
- G50（本地部署优势）

**关键金句**：
- "龙虾看不见，等于瞎养"
- "能说话不稀奇了。看不见，才是真的瞎"
- "先给它一双眼睛，能看见才能帮你干活"

### 25.10.3 第 3 件：给龙虾搭一座图书馆（知识库，能检索、能结构化）

**核心观点**：碎片记忆不够用，需要一座能分类、能检索、能长出结构的图书馆

**素材要点**：
- 第 11-37 天：搭建 collector + prism，整合进 memory-core
- 背景：有了记性和眼睛后，信息是碎片化的，跨领域问题翻不到
- **收集器（js-knowledge-collector）**：
  - 聊天里扔链接 → 抓全文 → AI 生成摘要和标签 → 存 SQLite
  - 覆盖十几个平台：微信、知乎、即刻、小红书、GitHub、Reddit、B 站、YouTube
  - 公开页面直连抓取，登录型平台调 js-eyes 复用浏览器登录态
  - inbox/batch 轮转：扔链接只入队，定时任务隔离会话批量处理（不卡对话）
  - 可选同步到 Flomo
  - 安装：clone 仓库 + 加载插件路径 + `openclaw knowledge setup-collector`
- **棱镜（js-knowledge-prism）**：
  - 把散乱素材拆成原子观点 → 归成观点组 → 蒸馏出高层结论
  - 三层蒸馏：journal → pyramid → outputs（灵感自《金字塔原理》）
  - 只处理增量，已处理的不重复碰
  - 安装：clone 仓库 + `openclaw.json` 的 `plugins.load.paths` 指向插件目录 + 配 AI API
  - CLI：`openclaw prism init/process/status/sync`
  - 可配定时任务自动转
- **整合进 memory-core**：
  - `memorySearch.extraPaths` 指向收集器摘要目录 + 棱镜高价值层目录
  - 棱镜挑食：只导出观点组、核心结论、视角背景（高密度），排除原始日记、原子表格、长文成品
  - 三源互补：collector 摘要 + prism 结构化洞察 + 原生个人记忆
- 验收标准：龙虾能跨源检索，串联不同来源的内容做结构化回答

## 叙事结构（文章大纲）

### 开场：后台留言

- 朋友的龙虾和豆包回答同一个问题，答案几乎一样
- 我的龙虾回答串联了公众号观点、个人笔记、GitHub 细节
- 区别不在龙虾本身，在你往里面喂了什么

### 转折：三件事，没有标准顺序

- 今天把这三件事交给你
- 你可以按顺序来，也可以从任何一件开始
- 不是"跟着我的节奏"，是"你的龙虾你的节奏"

### 论证：三件事

1. **第 1 件**：给龙虾记性（memory-core 插件，能记住）
2. **第 2 件**：给龙虾一双眼睛（js-eyes 浏览器插件，能看见）
3. **第 3 件**：给龙虾搭图书馆（collector + prism → memory-core，能检索、能结构化）

### 收尾：你的龙虾你的节奏

- 三件事，没有标准顺序
- 你的龙虾，你的节奏
- 第 1 件有记忆，第 2 件有眼睛，第 3 件有图书馆
- 开始养吧
