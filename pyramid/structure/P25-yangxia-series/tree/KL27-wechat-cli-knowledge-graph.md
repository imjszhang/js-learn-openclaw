**S (情境)**
上一篇 KL21，我讲了怎么用龙虾管理微信收藏的 900 篇文章。
文章发出去后，有读者问了一个很实在的问题：

**Q (问题)**

有。今天发现了一个好玩的新项目 wechat-cli——
它能直接解密微信数据库，把收藏、聊天记录全导出来，喂给龙虾。
你不用转发了。龙虾自己来拿。

### KL02 - wechat-cli：一个直接读你微信数据的新项目

**功能：工具引入**

**支撑**：
- 今天发现了一个好玩的新项目：wechat-cli
- 它做的事情很简单：直接从微信的加密数据库里读数据
- 微信把聊天记录存在 SQLCipher 加密的 SQLite 里，AES-256-CBC 逐页加密
- 但密钥在你自己电脑的进程内存里
- wechat-cli 用 320 行 C 代码扫描内存，提取密钥，逐页透明解密
- **纯本地、只读、零网络**——数据不离开你的电脑
- 不需要你手动转发，不需要你复制链接
- 龙虾够不到的数据，wechat-cli 帮你取出来

---

### KL03 - 11 条命令，把你的微信数据全释放出来

**功能：工具能力展示**

| 命令 | 功能 | 和龙虾的玩法 |
|------|------|-------------|
| export | 导出为 Markdown | 收藏夹/聊天记录一键导出，直接喂给龙虾 |
| favorites | 收藏夹查询 | 替代手动转发，批量导出收藏文章 |
| search | 关键词搜索 | 让龙虾定位你半年前的某条收藏 |
| history | 指定聊天历史记录 | 按群/按人导出讨论记录 |
| new-messages | 增量新消息 | 配合 cron 每天自动追加新收藏/新聊天 |
| stats | 群聊统计 | 分析哪个群产出价值最高 |
| sessions | 最近会话列表 | 快速了解最近的讨论动态 |
| contacts | 联系人搜索 | 按人检索历史对话 |

**核心突破**：
之前你给龙虾喂数据的方式是"手动转发"。
现在变成"龙虾自己去取"。
从推送模式变成拉取模式。

---

### KL04 - 收藏夹自动导出：解决 KL21 读者的那个问题

**功能：核心场景一（收藏夹）**

**支撑**：
- 用 `favorites` 命令查询收藏：
  - 按类型筛选：文章/图片/视频/文本
  - 按时间范围过滤
- 用 `export` 命令导出为 Markdown：
  - 自动带时间戳、来源信息
  - 干净的文本格式
- 导出后直接丢进 Prism 的 journal/ 目录
- 龙虾自动处理：atoms → groups → synthesis
- **不需要你一篇篇转发了**

**对比 KL21 的进化**：
| | KL21 方案 | KL27 方案 |
|---|---|---|
| 数据获取 | collector 自动抓取 + 手动转发 | wechat-cli 直接导出 |
| 用户操作 | 需要手动转发收藏 | 零操作，定时自动导出 |
| 覆盖范围 | 只能抓取公众号文章链接 | 收藏夹全部内容（含图片/视频/文本） |

### KL05 - 群聊暗知识：比收藏更有价值的东西

**功能：核心场景二（群聊暗知识）—— 延伸发现**

**支撑**：
- 解决收藏夹问题后，发现 wechat-cli 还能做更大的事
- 群聊里的"暗知识"：产品决策、技术选型、项目共识
- 这些从来没有被文档化，但价值不亚于正式文档
- 用 `export` 导出群聊 Markdown → Prism 处理：
  - Atoms：提取信息单元（"某某说 A 方案更好"）
  - Groups：归类（"架构选型讨论"、"技术踩坑记录"）
  - Synthesis：收敛观点（"团队最终倾向轻量方案"）
- **从"管理收藏"到"释放暗知识"——这才是 wechat-cli 的真正价值**
