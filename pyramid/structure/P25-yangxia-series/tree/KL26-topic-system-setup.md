# KL26 - 我是怎么用 OpenClaw +飞书多维表格，搭建养虾日记选题系统的

> 关联项目：养虾日记选题管理系统
> 素材来源：真实搭建过程

---

## SCQA

**S (情境)**  
养虾日记写到第19篇，有朋友问我：怎么做到不重复、不断更的？答案是我的"后台"——一个用 OpenClaw + 飞书多维表格搭建的选题管理系统。

**C (冲突)**  
但三个月前，我的选题管理还是一团糟：想法散落在 Flomo/Chrome/聊天记录里，进度完全不清楚，素材和产出脱节。

**Q (问题)**  
怎么搭建一个能支撑长期连载的选题管理系统？

**A (答案)**  
三层架构：飞书多维表格管状态、Prism 知识库存素材、OpenClaw 定时产出。把"我要写什么"的不确定性，变成"从列表里选一个"的确定性。

## Key Line 树

### KL01 - 选题混乱的三重困境（代入感）

**支撑**：
1. **想法散落**：Flomo/Chrome/聊天记录，想到哪记到哪

**钩子**：你是不是也这样？

---

### KL02 - 我的解决思路：三层架构（方案预览）

**支撑**：
```
飞书多维表格（管理层）← 状态、进度、优先级
        ↓
知识库 Prism（素材层）← Journal、Atoms、Synthesis
        ↓
定时产出（执行层）← 自动检测、自动生成
```

**核心洞察**：不是用一个工具解决所有问题，是每个工具做最擅长的事

---

### KL03 - 飞书多维表格设计（实操）

**支撑**：
核心字段：
- 篇号（KLxx）
- 标题
- 核心痛点 / 读者心理 / 解决方案
- 选题类别（5大类）
- 优先级（P0/P1/P2）
- 状态（待写/进行中/已完成/已发布）
- 飞书文档链接

**效果截图/描述**：25个选题清晰管理

---

### KL04 - 与知识库的关联设计（关键连接）

**支撑**：
- 表格管"状态"，知识库管"内容"
- Journal 路径：`journal/YYYY-MM-DD/klxx-slug.md`
- KL 框架：`pyramid/structure/P25-yangxia-series/tree/KLxx-slug.md`
- 产出：`outputs/yangxia-series/xx.md`

**关键设计**：状态和内容分离，表格只记录"在哪里能找到"

---

### KL05 - 定时产出绑定（自动化）

**支撑**：
```bash
knowledge_prism_bind_output \
  --perspectiveDir P25-yangxia-series \
  --template yangxia-series \
  --klStrategy manual
```

**效果**：结构变化自动触发产出，不用手动惦记
