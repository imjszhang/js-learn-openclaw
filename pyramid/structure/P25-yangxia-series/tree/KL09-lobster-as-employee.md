# KL09: 别把龙虾当工具，把它当员工——OpenAI 管这叫 Harness Engineering

> 所属视角：P25-yangxia-series
> 上层论点：通过 37 天真实"养虾"记录，展示从看到 Moltbook 上 AI 社交很火、养出第一只社交龙虾 JS_BestAgent 开始，到建立生态导航站并养出第二只管理龙虾 JS_CLAW，再到衍生出浏览器控制（js-eyes）、知识收集（js-knowledge-collector）、知识结构化（js-knowledge-prism）三个工具项目的完整路径。不是教程，是一个普通人从零到构建出自己的 AI 工具小生态的成长日记。

## 支撑论点

### 25.9.1: 同一个控制论模式，在历史上出现了三次
- 逻辑顺序：递进
- 引用 atoms: P25-01
- 引用 groups: G01

从瓦特的离心调速器（1788），到 Kubernetes 的控制器，再到 OpenAI 的 Harness Engineering——工程师的角色反复完成同一种转变：从亲手操作系统，变成设计让系统自动运转的机制。维纳在 1948 年将这种模式命名为控制论（Cybernetics）。

### 25.9.2: Agent 失败的根因是知识外化失败，不是能力不足
- 逻辑顺序：因果
- 引用 atoms: P25-02, P25-03
- 引用 groups: G02

OpenAI 的核心洞察：agent 反复犯同样的错，不是因为模型不够强，而是因为工程师把判断标准（什么叫"好"、架构鼓励哪些模式、刻意回避哪些模式）锁在脑子里从未写出来。不外化这些知识，agent 第 100 次犯的错和第 1 次一模一样。

### 25.9.3: OpenClaw 的六份 bootstrap 文件就是你的 Harness
- 逻辑顺序：结构
- 引用 atoms: P25-04, P25-05, P25-06, P25-07
- 引用 groups: G03

OpenClaw 在每个 agent 工作区中通过六份 Markdown 文件（AGENTS.md / SOUL.md / USER.md / IDENTITY.md / HEARTBEAT.md / TOOLS.md）实现知识外化。每次 agent 启动时从磁盘注入上下文，相当于新员工入职的全套材料：员工手册、企业文化准则、老板档案、工牌、值班表、工位设备清单。配齐这六份文件，龙虾从通用聊天机器人变成"你的员工"。

## 论点间关系

本 Key Line 遵循"为什么 → 是什么 → 怎么做"的递进逻辑：先用三次历史重复建立"控制论"的认知框架（25.9.1），再用 OpenAI 的洞察揭示"知识外化"是 agent 可靠运转的关键前提（25.9.2），最后落地到 OpenClaw workspace bootstrap 文件体系作为具体实现（25.9.3）。
