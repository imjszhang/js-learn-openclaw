# KL13: OpenClaw 好用的关键：配置好记忆

> 一句话：普通 AI 的"记忆"是聊完就忘的短期缓存；三层记忆系统是一个会自我生长的数字大脑——原生记忆保偏好、Prism 做提炼、Collector 扩视野。

---

## 支撑论点

### 论点 1：原生记忆层——你的偏好和决策，只有它真正"记得"

**核心内容**：
- `MEMORY.md` + `memory/*.md` 构成当前会话的引导文件
- 不是简单的"聊天记录"，而是人工筛选的精要
- 包含：个人决策、偏好设定、待办事项、每日日志

**关键数据**：
- 混合搜索：向量 70% + BM25 30%
- MMR 重排序 + 30 天半衰期时间衰减
- 配置：`jina-embeddings-v5-text-small` 嵌入模型

**引用素材**：
- `MEMORY.md` 中的技术配置段落
- `memory/2026-03-26.md` 的 KL12 选型讨论

---

### 论点 2：Knowledge Prism——把信息洪流消化成结构化知识

**核心内容**：
- 不是存储，是"消化"：Journal → Atoms → Groups → Synthesis → Outputs
- 金字塔结构，任何结论都可追溯到原始日记
- 层次筛选：只导出高价值 Groups，降低 80% 噪音

**关键数据**：
- 60 个 Groups（G01-G60+）
- 2 个注册知识库（cyber-taoist + OpenClaw 个人实践）
- 全自动化管线：process → agent_index → output

**引用素材**：
- KL06（知识棱镜）的"消化系统"比喻
- G57（Prism 记忆桥接）
- `P23-practice-diary-context.md` 的金字塔结构描述

### 论点 3：Collector——1289 篇外部情报的聚合与检索

**核心内容**：
- 9 大平台文章自动抓取 + AI 总结
- 每篇包含：概要 + 摘要 + 推荐理由
- inbox/batch 轮转架构，不阻塞主会话

**关键数据**：
- 1289 篇入库（微信 908 + GitHub 189 + 知乎 130 + 其他）
- 自动化：Flomo 推送 + Prism 导出（可选）
- 检索：关键词/平台/时间范围多维搜索

**引用素材**：
- KL05（知识收集器）的"感官系统"比喻
- G52（Collector 架构设计）

---

### 论点 4：三层协同——1+1+1 > 3 的认知增强效应

**核心内容**：
- 场景 1：回答技术问题 → 三层联合检索
- 场景 2：写文章 → 融合个人经验 + 结构化知识 + 外部情报
- 场景 3：日常对话 → 精准回忆具体细节

**关键洞察**：
- 原生记忆解决"你是谁"
- Prism 解决"你知道什么"
- Collector 解决"世界发生了什么"
- 三者协同 = 完整的认知图景

---

### 论点 5：时间复利——这是越久越值钱的数字资产

**核心内容**：
- 普通 AI：每次对话从零开始
- 三层记忆：每次对话都站在过去 N 天的积累之上
- 不可复制性：个人历史 + 知识提炼方式 + 情报库 = 独特资产

**金句预埋**：
- "记忆不是存储，是消化；不是仓库，是土壤"
- "你的龙虾比你更'了解'你自己的知识"

## 逻辑顺序

1. **引入**：对比测试——三个 AI 回答同一个问题，只有我的龙虾答对（悬念）
2. **第一层**：原生记忆——为什么它能"记得"我的偏好
3. **第二层**：Prism——为什么它能"理解"我的知识体系
4. **第三层**：Collector——为什么它能"掌握"外部情报
5. **协同**：三层如何配合解决真实场景问题
6. **对比**：与 ChatGPT/Claude/普通 RAG 的方案对比
7. **升华**：时间复利与不可复制的数字资产
8. **结尾**：金句 + 开源声明
