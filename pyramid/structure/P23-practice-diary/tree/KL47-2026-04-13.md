# KL73: 第 73 天：暗知识零信任自动化链路构建

> 所属视角：P23-practice-diary
> 上层论点：必须实施“架构分层分离 + 全链路自动化闭环 + 纵深安全防御”三位一体战略：以五层抽象网关为基石，通过双 Cron 分段驱动与 inbox/batch 轮转实现无人值守的知识产出，同时严格执行执行审批、权限收紧及记忆职责剥离，确保系统从“可记录”平稳演进为“可教学”的智能体。

## 支撑论点

### 73.1: 构建“本地内存解密-LLM 实体提取 - 图谱可视化”的零信任自动化链路，将高价值群聊对话转化为可检索的结构化资产
- 逻辑顺序：结构
- 引用 atoms: WK-01, WK-07, WK-14
- 引用 groups: G99

针对微信群聊中未被文档化但极具价值的产品决策与技术共识（暗知识），本方案摒弃传统人工整理，转而采用纯本地工具扫描进程内存提取加密数据，结合 LLM 语义分析与图谱算法，实现从碎片化聊天到结构化知识资产的自动转化。

### 73.2: 依托轻量级 wechat-cli 与 Graphify 工具链，实现从加密 SQLite 到交互式知识图谱的端到端数据流转
- 逻辑顺序：时间
- 引用 atoms: WK-02, WK-05, WK-06
- 引用 groups: G99

利用仅由 320 行 C 代码构成的 wechat-cli 在 macOS 本地解密 AES-256 加密数据库并导出带元数据的 Markdown，随后由 Graphify 接收标准输入，通过 LLM 提取实体关系及 Leiden 算法聚类，最终输出可交互的 HTML 图谱与分析报告。

### 73.3: 建立双 Cron 分段驱动的无人值守运维机制，并严格执行安全合规与平台限制以确立零信任边界
- 逻辑顺序：程度
- 引用 atoms: WK-08, WK-11, WK-12
- 引用 groups: G99

通过 cron 定时任务配合 `new-messages` 命令实现每日自动增量同步，同时在涉及内存扫描与 root 权限的操作中明确安全警示，并严格限定 macOS 平台适用范围，确保自动化流程在合规与可控的前提下运行。

## 论点间关系

子论点遵循“价值定义→技术实现→运维保障”的结构逻辑：首先明确暗知识挖掘的业务目标，接着阐述具体的工具链数据流转过程，最后界定自动化运行的安全边界与约束条件，共同构成完整的零信任自动化闭环。
