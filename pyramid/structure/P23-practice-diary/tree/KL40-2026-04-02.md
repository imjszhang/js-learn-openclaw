## 支撑论点

### 62.1: 发布到 ClawHub 必须采用"Plugin 形态 + 绝对路径 + 语义化版本"策略以规避技能能力限制与 Windows 路径解析陷阱
- 逻辑顺序：结构
- 引用 atoms: PK-01, PK-02, PK-04, PK-05, PK-06, PK-08, PK-09

鉴于 js-knowledge-prism 核心依赖工具注册、CLI 及 HTTP 路由等执行能力，必须将其定义为含 `openclaw.plugin.json` 的 Plugin 而非纯指令 Skill，并随附 SKILL.md 作为使用指南。在执行发布时，需严格使用绝对路径以绕过 Windows 环境下相对路径导致的文件缺失误报，同时遵循语义化版本规范确保在 ClawHub 原生注册表中的优先安装权。
