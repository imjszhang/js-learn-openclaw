# G95: 养虾日记选题必须聚焦“用户与 JS_CLAW 共同进化”的实证轨迹，以 Harness 工程学重构知识生长逻辑

> 养虾日记系列的核心价值不在于工具横评，而在于通过 800+ 次对话记录与 Journal 闭环，实证 Harness 作为活系统如何让用户摆脱对大模型的绝对依赖并实现双向塑造。

## 包含的 Atoms

| 编号  | 来源                     | 内容摘要 |
| ----- | ------------------------ | -------- |
| KT-01 | kl25-topic-selection     | 养虾日记系列的核心选题方向应聚焦于真实使用 OpenClaw 的个人体验，而非单纯的横评或对比 |
| KT-02 | kl25-topic-selection     | Karpathy 知识库工作流的核心洞察是知识库由原始素材经 LLM 自动“生长”而成，而非人工整理 |
| KT-03 | kl25-topic-selection     | Harness 工程学的核心论点是 Harness 作为活系统可自我迭代，让使用者摆脱对大模型的绝对依赖 |
| KT-04 | kl25-topic-selection     | 用户与 JS_CLAW 的 800+ 次对话记录构成了共同进化的轨迹，双方存在双向塑造关系 |
| KT-05 | kl25-topic-selection     | 核心逻辑链将 Karpathy 的 raw/目录映射为用户与 JS_CLAW 的对话记录，将 LLM 自动生长映射为 Journal 到文章的转化 |
| KT-06 | kl25-topic-selection     | Journal 功能的本质是构建自我强化的反馈循环：对话产生 Journal，JS_CLAW 读取后承接上下文产生新洞察 |
| KT-07 | kl25-topic-selection     | 技术排查场景的标准流程为：提出问题→讨论解决→记录 Journal→下次遇到时直接召回 |
| KT-08 | kl25-topic-selection     | 知识积累场景的标准流程为：阅读文章→与 JS_CLAW 讨论→记录 Journal→Prism 提取→生长为文章 |
| KT-09 | kl25-topic-selection     | 文章开头应采用“对话/龙虾回复截取”结合“热点嫁接”的方式，以区分于已使用的开头类型 |
| KT-10 | kl25-topic-selection     | KL25 选题的数据支撑包括 800+ 次对话记录、1289 篇入库文章及 60 个知识分组 |
| KT-11 | kl25-topic-selection     | 在养虾日记系列中禁止使用"AI"泛称主角，必须使用具体的龙虾名字（如 JS_CLAW）以维持人设一致性 |
| KT-12 | kl25-topic-selection     | 写日记的目的不是记录过去，而是构建用户与 JS_CLAW 共同进化的循环证据 |

## 组内逻辑顺序

逻辑顺序遵循“选题定位→理论映射→机制详解→实操规范→数据支撑”的结构：
1. **选题定位 (KT-01, KT-03)**：明确系列核心是个人体验与 Harness 工程学，而非工具对比。
2. **理论映射 (KT-02, KT-04, KT-05)**：将 Karpathy 理论与用户实际数据（对话记录）进行映射，确立“共同进化”论点。
3. **机制详解 (KT-06, KT-07, KT-08)**：深入解析 Journal 反馈循环在技术排查与知识积累两种场景下的具体运作流程。
4. **实操规范 (KT-09, KT-11)**：规定文章开头写法与人设称呼禁忌，确保内容风格统一。
5. **数据支撑与升华 (KT-10, KT-12)**：列举具体数据证明可行性，并升华写日记的终极目的。
