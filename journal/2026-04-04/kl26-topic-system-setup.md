## 1. 背景与动机

KL19 发布后，有朋友问我：养虾日记写了19篇，怎么做到不重复、不断更的？

答案是我的"后台"——一个用 OpenClaw + 飞书多维表格搭建的选题管理系统。

这篇就来完整复盘这个系统的搭建过程。

---

## 2. 痛点：选题混乱的三重困境

在搭这个系统之前，我的选题管理是一团糟：

**第一重：想法散落各处**
- 突然想到一个选题 → 记到 Flomo
- 看到一篇好文章 → 收藏到 Chrome
- 和朋友聊天有启发 → 忘在聊天记录里

**第二重：进度不清楚**
- 这篇写到哪了？素材够了吗？
- 下一篇要写什么？完全没概念
- 哪些选题已经过时了？不知道

**第三重：素材和产出脱节**
- journal 记了一堆，但不知道能产出什么
- prism 结构更新了，但产出没跟上
- 飞书文档和知识库各管各的

---

## 3. 方案设计：三层架构

解决思路：把选题管理拆成三层，每层用最适合的工具

```
飞书多维表格（管理层）
    ↓ 选题状态、进度、优先级
知识库 Prism（素材层）
    ↓ Journal、Atoms、Groups、Synthesis
定时产出（执行层）
    ↓ 自动检测变化、自动生成文章
```

---

## 4. 搭建过程

### 4.1 飞书多维表格设计

核心字段：
- 篇号（KLxx）
- 标题
- 核心痛点
- 读者心理
- 解决方案
- 选题类别（解决痛点/疑问新知/避坑指南/盲区纠正/轻松好玩）
- 优先级（P0/P1/P2）
- 状态（待写/进行中/已完成/已发布）
- 飞书文档链接

### 4.2 与知识库的关联

关键设计：表格里只管理"状态"，具体内容在知识库里

- Journal 路径：`journal/YYYY-MM-DD/klxx-slug.md`
- KL 框架路径：`pyramid/structure/P25-yangxia-series/tree/KLxx-slug.md`
- 产出路径：`outputs/yangxia-series/xx.md`

### 4.3 定时产出绑定

```bash
# prism 自动检测结构变化，自动产出
knowledge_prism_bind_output \
  --perspectiveDir P25-yangxia-series \
  --template yangxia-series \
  --klStrategy manual
```

### 4.4 工作流闭环

```
多维表格选题 → 知识库Journal → KL框架 → 产出文章
     ↑                                    ↓
     └──────────── 更新状态 ← 发布到公众号
```

## 6. 关键洞察

**不是用一个工具解决所有问题，是让每个工具做最擅长的事**

- 飞书多维表格：状态管理、进度可视化
- Prism 知识库：深度素材、结构化思考
- OpenClaw Agent：自动化执行、定时产出

**选题系统的本质是：把"我要写什么"的不确定性，变成"从列表里选一个"的确定性**
