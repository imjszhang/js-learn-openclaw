## 3. 方案设计

### 3.1 KL10 核心论点

**标题**：让你的龙虾和别的 AI 不一样：装好 OpenClaw 一定要做的三件事

**核心公式**：
```
有记忆 → 有眼睛 → 有图书馆
```

**关键金句**：
- "37 天里我其实就做了三件事"
- "你可以按顺序来，也可以从任何一件开始"
- "三件事，没有标准顺序——你的龙虾，你的节奏"
- "龙虾看不见，等于瞎养"
- "没有记忆的龙虾，每次对话都是第一次见面"
- "收集器管进货，棱镜管加工，memory-core 管上架"

### 3.2 叙事结构

| 章节 | 内容 | 前篇承接 |
|------|------|----------|
| 开场 | 对比测试：朋友的龙虾 vs 我的龙虾 vs 豆包 | 社交场景反转 |
| 转折 | 37 天里我其实就做了三件事 | 个人实践总结 |
| 论证 1 | 第 1 件：给龙虾记性（memory-core 插件） | 官方插件 |
| 论证 2 | 第 2 件：给龙虾一双眼睛（浏览器插件） | KL03 |
| 论证 3 | 第 3 件：给龙虾搭图书馆（collector + prism → memory-core） | KL05 + KL06 |
| 收尾 | 三件事，没有标准顺序 | 呼应 SCQA |

### 3.3 开头类型设计

**类型**：对比测试 + 反转

**首句**："上周吃饭，朋友掏出手机，让他的龙虾和豆包回答同一个问题。"

**避免撞型**：与已用 9 种开头类型对比（时间 + 画面/热点新闻/AI 交互失败/读者反馈/翻旧东西 + 数字/龙虾交互/热点人物/官方发布/工作场景），本篇为「对比测试 + 反转」，不撞型
