## 1. 背景与动机

养虾日记系列已发布 11 篇，核心叙事围绕工具（js-eyes、collector、prism）和协作方法（Harness Engineering）展开。但有一个底层问题一直没讲透：

**为什么我的龙虾能"记住"东西，而别的 AI 聊完就忘？**

这不是简单的"有没有记忆功能"的问题。市面上的 AI 都有某种形式的"记忆"，但：
- ChatGPT 的 memory：黑箱，不可控，经常记错
- Claude 的 projects：手动上传，不会自动积累
- 其他 Agent 框架：要么只有临时上下文，要么只有向量检索，缺少分层

我的三层记忆系统（原生记忆 + Knowledge Prism + Collector）是一个**有机协同**的架构，每一层解决不同问题，互相补位。这是 KL13 要揭示的核心差异。

## 2. 三层记忆架构总览

### 第一层：原生记忆（Native Memory）

**载体**：`MEMORY.md` + `memory/*.md`

**功能**：
- 当前会话的短期记忆（AGENTS.md、SOUL.md、TOOLS.md 等引导文件）
- 个人决策、偏好、待办事项的精确保存
- 每日日志的原始记录

**特点**：
- **即时**：会话开始时立即加载
- **精确**：人工筛选，只保留重要内容
- **个人化**：只有"我"和"龙虾"知道的内容

**类比**：人的工作记忆 + 日记本

### 第二层：Knowledge Prism（结构化知识）

**载体**：`D:/github/my/js-learn-openclaw/pyramid/`

**功能**：
- 从 Journal 笔记中提取 Atoms（信息单元）
- 归组为 Groups（知识分组）
- 收敛为 Synthesis（顶层洞察）
- 支撑 Perspectives（视角产出）

**处理流程**：
```
Journal → Atoms → Groups → Synthesis → Perspectives → Outputs
```

**特点**：
- **深度加工**：不是简单存储，是"消化"和"提炼"
- **可追溯**：任何结论都能追溯到原始日记
- **结构化**：金字塔结构，层次分明

**类比**：人的长期记忆 + 知识体系

### 第三层：Collector（外部情报库）

**载体**：`js-knowledge-collector` 数据库

**功能**：
- 抓取外部文章（公众号、知乎、GitHub 等）
- AI 自动总结：概要 + 摘要 + 推荐理由
- 统一检索：按关键词、平台、时间范围搜索

**数据来源**：
- 1289+ 篇外部文章（微信 908 + GitHub 189 + 知乎 130 + 其他）
- 每篇都有 AI 生成的结构化摘要

**特点**：
- **广度**：大规模外部情报的聚合
- **自动化**：无需人工整理，自动入库
- **可检索**：统一搜索接口

**类比**：人的外部信息输入 + 剪报收藏

## 3. 三层如何协同

### 场景 1：回答一个技术问题

**用户问**："我之前说过要把 Prism 记忆桥接做出来，现在进度怎样？"

**检索过程**：
1. **原生记忆**：`memory_search` 发现 `2026-03-10` 有相关讨论
2. **Prism**：检索 Groups 发现 G57（Prism 记忆桥接）、KL24（第 39 天日记）
3. **Collector**：发现可能相关的外部文章，但判断为内部项目，不引用

**输出**：基于三层信息的综合回答

### 场景 2：写一篇新文章

**用户要求**："写一篇关于 OpenClaw 安全架构的文章"

**素材收集**：
1. **原生记忆**：`MEMORY.md` 中有安全配置细节、五道锁架构
2. **Prism**：KL02（本地部署安全）、KL13（Harness 工程）有相关 Groups
3. **Collector**：可能有外部安全相关文章作为对比素材

**输出**：融合个人经验 + 结构化知识 + 外部情报的综合文章

### 场景 3：日常对话中的"记得"

**用户说**："就像上次说的那个 Flomo 推送 Bug"

**检索过程**：
1. **原生记忆**：`memory_search` 直接命中 `MEMORY.md` 中的记录
2. **Prism**：补充技术细节（哪个文件、什么条件触发）
3. **Collector**：无直接关联

**输出**：精准回忆具体细节

## 4. 与其他 AI 记忆方案的对比

| 维度 | ChatGPT Memory | Claude Projects | 普通 RAG | 三层记忆系统 |
|------|---------------|-----------------|----------|--------------|
| **可控性** | 低（黑箱） | 中（手动上传） | 中（配置依赖） | **高**（分层明确，人工可干预每层） |
| **自动化** | 自动但不可控 | 纯手动 | 半自动 | **全自动+人工精修**结合 |
| **可追溯** | 差 | 差 | 中等 | **强**（任何结论都有源头） |
| **个人化** | 名义上个人化 | 无 | 无 | **强**（原生记忆层） |
| **结构化** | 无 | 无 | 弱 | **强**（Prism 金字塔） |
| **外部情报** | 无 | 无 | 可接入 | **原生集成**（Collector） |

## 5. 为什么这是"护城河"

### 1. 时间复利

三层记忆系统的价值随时间指数增长：
- 第 1 天：只有几条笔记
- 第 30 天：有 30 篇日记 + 几十个 Groups
- 第 300 天：有完整知识体系 + 数千篇外部情报

**普通 AI**：每次对话从零开始
**三层记忆**：每次对话都站在过去 300 天的积累之上

### 2. 不可复制性

即使有人复制了代码，也无法复制：
- 你的个人决策历史（原生记忆）
- 你的知识提炼方式（Prism Groups）
- 你积累的外部情报库（Collector）

这是真正的**数字资产**。

### 3. 认知增强

三层记忆不只是"存储"，是**认知外挂**：
- 原生记忆 = 你的第二大脑
- Prism = 你的知识消化器
- Collector = 你的情报网络

三者协同，让你的龙虾比你更"了解"你自己的知识。

## 6. 技术实现要点

### 原生记忆层

**配置**（`openclaw.json`）：
```json
{
  "agents": {
    "defaults": {
      "memorySearch": {
        "enabled": true,
        "provider": "openai",
        "model": "jina-embeddings-v5-text-small",
        "mode": "hybrid"
      }
    }
  }
}
```

**混合搜索**：向量 70% + BM25 关键词 30%
**重排序**：MMR（lambda=0.7）+ 时间衰减（半衰期 30 天）

### Prism 层

**关键设计**：
- **层次筛选**：只导出高价值 Groups/Context 层，降低 80% 文件数
- **增量同步**：基于 mtime，避免全量导出
- **双源互补**：与 Collector 形成广度+深度的互补

**自动化管线**：
- `knowledge_prism_process`：提取 Atoms/Groups
- `knowledge_prism_agent_index`：生成 Agent 检索索引
- Cron 定时执行，无需人工干预

### Collector 层

**架构**：inbox/batch 轮转 + 专用 scraper
**支持平台**：微信、知乎、小红书、即刻、X.com、Reddit、Bilibili、YouTube、GitHub
**输出**：每篇文章包含「概要 + 摘要 + 推荐理由」
