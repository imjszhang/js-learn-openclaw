# 教你的龙虾，用你的 flomo：从被动记录到主动思考的进化

> Day 77 · 2026-04-17 · 骨架 [KL49](./P23-practice-diary/KL49-2026-04-17.md)

今天终于解决了 flomo 用户最深层的痛点：笔记记了却读不完。当搜索一个关键词跳出 200 条结果时，大多数人只会关掉窗口，而我决定让系统替我们先读，把“堆积”转化为真正的“认知升维”。

## 系统定位：从“够不到数据”到“替你思考”

之前的 KL27 解决了微信数据“龙虾够不到”的问题，通过 wechat-cli 让系统能自动抓取数据。但今天面对 flomo，我发现新的瓶颈不是获取，而是消化。flomo 的核心困境在于“记太多读不过来”：人脑无法同时处理数百条笔记间的隐藏关联，导致大量素材沉睡。

因此，`js-knowledge-flomo` 项目的定位发生了根本性转变：它不再只是一个维护工具，而是一个替代人工阅读的自动化分析引擎。这标志着养虾日记系列从 KL30 的“系统自生长”正式进化为 KL31 的“系统替你思考”。我们的目标很明确：除非系统替你先读并告诉你发现了什么，否则记了不读等于白记。

## 五层模块化架构与多端协同

为了支撑这一目标，我基于 Node.js 18+、SQLite 本地缓存和 OpenAI 兼容 API 构建了轻量级的五层架构。项目同时扮演 MCP Client 和 MCP Server 双重角色，确保了极高的灵活性。

核心代码结构清晰分层：`cli/` 目录承载 auth、search 及核心的 `processor.js` AI 加工引擎；`mcp-server/` 提供标准的 AI 分析与笔记读写工具；`prompts/` 目录独立管理洞察、关联、大纲等提示词模板，实现逻辑与策略解耦；此外还包含了 `openclaw-plugin` 技能包和 `src/` 下的 Web UI 静态资源。这种设计让我们能同时支持四种使用场景：在 OpenClaw 中通过 15 个专用工具对话式操作、在终端运行独立 CLI、在 Cursor 或 Claude Desktop 中作为 MCP 服务调用，以及启动本地 Web UI 进行可视化洞察生成。

## 七大 AI 分析功能：挖掘隐藏关联与观点演变

系统的核心价值在于其七大 AI 分析功能，它们将散乱的时间线笔记转化为结构化知识。

首先是**洞察发现**，系统默认拉取 50 条笔记缓存至 SQLite，利用 LLM 分析反复主题、情绪变化趋势及跨标签的隐藏关联，直接回答“你最近真正在关心什么”。其次是**关联发现**，强制 LLM 跨时空寻找至少 3 组有意义的连接，推测用户潜意识里在思考的大问题。针对特定话题，**观点演变**功能能按时间线标注关键转折点，清晰展示立场从 A 到 B 的证据链。

在产出侧，**写作大纲**功能拒绝凭空生成，而是基于真实的散落笔记构建结构化大纲；**素材搜集**则结合关键词搜索与 flomo 官方的“相关推荐”接口，确保素材的全面性。针对日益混乱的标签体系，**标签建议**自动为无标签笔记归类，而**标签审计**则深入分析标签树的重复、命名不一致及层级问题，给出具体的合并或重命名建议。这些功能共同实现了从“搜索”到“发现”的跨越。

## 安全隐私原则：数据不出用户环境

在构建自动化闭环的同时，我严格执行了安全纵深防御策略。接入层坚持使用 flomo 官方 MCP 协议，通过标准的 OAuth 授权避免逆向工程带来的安全风险。数据存储层选用 SQLite，利用其轻量、零依赖且支持 SQL 查询的特性构建本地缓存，确保原始数据完全掌握在用户手中。

最关键的是 AI 分析环节，所有 LLM 调用均指向用户自建的 API 接口，严格保证数据不出用户环境。同时，我将所有 Prompt 模板独立存放在 `prompts/` 目录下，这不仅便于单独优化提示词工程，更实现了业务逻辑与 AI 策略的彻底解耦，让系统既安全又可迭代。

## 今天的收获

- **痛点演进识别**：知识管理的瓶颈已从“数据获取难”（KL27）转移为“信息消化难”（KL31），系统定位需从“自动维护”升级为“主动思考”。
- **架构双重角色**：采用 MCP Client + Server 双角色架构，配合 SQLite 本地缓存，能同时满足 CLI、IDE 插件及 Web UI 的多端需求，最大化复用性。
- **功能深度设计**：AI 分析不能仅停留在摘要，必须深入挖掘“观点演变”、“跨时空关联”及“标签审计”等高维认知任务，才能替代人工阅读。
- **安全隐私底线**：坚持 OAuth 官方协议、本地缓存及自建 LLM 接口三位一体，是构建可信个人知识系统的必要前提。
- **策略代码解耦**：将 Prompt 模板独立目录化管理，是确保 AI 应用可长期维护和快速迭代的关键工程实践。
