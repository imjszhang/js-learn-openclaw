命令立即报错，提示"SKILL.md required"，尽管该文件明明就在当前目录下。经过排查，确认这是 ClawHub CLI 在 Windows 下对相对路径 `.` 解析存在缺陷所致。解决方案是改用完整的绝对路径：
```bash
clawhub publish "D:/github/my/js-knowledge-prism" \
  --slug js-knowledge-prism \
  --name "JS Knowledge Prism" \
  --version 1.8.0 \
  --tags latest \
  --no-input
