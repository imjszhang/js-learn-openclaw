## 第一部分：在中国镜像安装技能（消费者视角）

打开中国镜像站：**cn.clawhub-mirror.com**

页面很简洁：精选 Skill、热门 Skill、一键安装命令。

比如我想安装 js-eyes（浏览器自动化技能），直接复制首页的命令：

```bash
npx clawhub@latest install js-eyes --registry=https://cn.clawhub-mirror.com
```

或者让龙虾自己装：

> 「通过 clawhub 寻找 js-eyes 技能，registry 使用中国镜像站 https://cn.clawhub-mirror.com」

几秒后，技能安装完成。现在我的龙虾有了眼睛，可以控制浏览器、解决登录态难题、抓取网页数据。

**关键认知**：中国镜像与主站内容实时同步，只是访问更快。你不需要担心「镜像版是不是阉割版」——它是完整的，只是物理位置更近。

**推荐安装的入门技能组合**：

| 技能 | 功能 | 为什么先装它 |
|------|------|-------------|
| js-eyes | 浏览器自动化 | 解决 90% 的登录态和网页抓取问题 |
| knowledge-collector | 知识收集 | 让龙虾能「看到」互联网上的文章 |
| knowledge-prism | 知识结构化 | 把收集的信息变成可复用的洞察 |
| comfyui-client | AI 图像生成 | 本地生成配图，不依赖云端 |

这四个技能装完，你的龙虾已经从「聊天机器人」进化成「有感官、有大脑、有创造力」的助手。

## 第二部分：本地切换镜像源（进阶用户视角）

如果你之前已经配置了主站，想切到中国镜像，很简单。

**核心命令**（npm 方式）：

```bash
npx clawhub@latest install <skill-name> --registry=https://cn.clawhub-mirror.com
```

**其他包管理器**：

```bash
# pnpm
pnpm dlx clawhub@latest install <skill-name> --registry=https://cn.clawhub-mirror.com

# bun
bunx clawhub@latest install <skill-name> --registry=https://cn.clawhub-mirror.com
```

**实际示例**：安装 js-x-monitor（X.com 监控技能）

```bash
npx clawhub@latest install js-x-monitor --registry=https://cn.clawhub-mirror.com
```

安装完成后，OpenClaw 会自动加载技能。你可以通过 `/status` 或询问龙虾来确认技能已激活。

**什么时候需要切换镜像？**

- 主站访问慢或不稳定
- 在中国大陆网络环境下
- 想要更快的技能安装速度

**注意**：切换镜像不影响已安装的技能，只是后续安装会走中国镜像源。

## 第三部分：制作你的第一个技能（生产者视角）

### 为什么要做技能？

我用 OpenClaw 30 天后，发现了一个规律：

**80% 的重复工作，都可以封装成技能。**

比如：
- 每天用 ComfyUI 生成配图 → 封装成 js-comfyui-client
- 每天监控 X.com 大 V 动态 → 封装成 js-x-monitor
- 每天解决浏览器登录态 → 封装成 js-eyes

把个人脚本变成技能，有三个好处：

1. **可复用**：一次制作，多处安装
2. **可分享**：社区共建，互惠互利
3. **可维护**：版本管理，迭代更新

### 最小可运行技能的结构

一个技能只需要三个文件：

```
my-skill/
├── SKILL.md          # 核心定义：技能是做什么的、怎么用
├── tools/            # 工具实现：具体功能代码
│   └── myTool.js
└── assets/           # 资源文件：图片、配置模板等
    └── config.json
```

**SKILL.md 是核心**。它告诉 OpenClaw：
- 这个技能解决什么问题
- 提供了哪些工具
- 工具怎么调用
- 使用示例是什么

### 使用 Skill Creator 工具

OpenClaw 内置了 Skill Creator，可以帮你快速创建技能骨架。

安装 Skill Creator：

```bash
npx clawhub@latest install skill-creator --registry=https://cn.clawhub-mirror.com
```

然后对龙虾说：

> 「帮我创建一个技能，名字叫 js-weather，功能是查询实时天气」

Skill Creator 会：
1. 创建技能目录结构
2. 生成 SKILL.md 模板
3. 提供工具代码示例
4. 给出测试建议

你只需要填充业务逻辑，不用从零写脚手架。

## 第四部分：上传技能到官方市场（发布者视角）

### 提交流程

1. **开发**：本地完成技能开发和测试
2. **文档**：确保 SKILL.md 完整、清晰
3. **打包**：整理目录结构
4. **提交**：通过 ClawHub 网页表单提交
5. **审核**：官方进行基础安全扫描
6. **上线**：审核通过后上架

**提交地址**：clawhub.ai/submit（或中国镜像站的对应入口）

### 审核标准

根据我的三次上架经验，审核关注这些点：

| 维度 | 要求 | 我的案例 |
|------|------|----------|
| 安全 | 无恶意代码、权限声明清晰 | js-eyes 需说明浏览器控制权限 |
| 功能 | 工具单一、职责明确 | js-comfyui-client 只负责调用，不处理提示词 |
| 文档 | SKILL.md 完整、有使用示例 | js-x-monitor 需说明 X.com 数据获取方式 |
| 协议 | 开源协议明确 | 推荐 MIT 协议 |

### 我的上架经验

**js-eyes（第一个技能）**
- 审核反馈：需要更详细的浏览器权限说明
- 解决：在 SKILL.md 中增加「权限与安全」章节

**js-comfyui-client**
- 审核反馈：ComfyUI 依赖需要明确说明
- 解决：添加「前置要求」章节，说明需本地安装 ComfyUI

**js-x-monitor（最新）**
- 审核反馈：X.com 数据获取需合规说明
- 解决：说明使用公开 API、不涉及账号破解

**核心认知**：审核不是为了卡你，是为了保护用户。把「潜规则」变成「显式规则」，技能质量反而提升。

## 第五部分：我的三个技能与开发心得

### 已上架技能清单

| 技能 | 功能 | 使用场景 | 上架时间 |
|------|------|----------|----------|
| **js-eyes** | 浏览器自动化 | 解决登录态、网页抓取、自动化操作 | 2026-03 |
| **js-comfyui-client** | ComfyUI 图片/视频生成 | 本地 AI 绘画、图生视频、批量生成 | 2026-03-25 |
| **js-x-monitor** | X.com 账号动态监控 | 追踪大 V 推文、热点捕捉、信息收集 | 2026-03-31 |

### 每个技能的故事

**js-eyes（第一个技能）**

起源：我想让 OpenClaw 读取我收藏的公众号文章，但登录态搞不定。

解决：写了一个浏览器扩展做桥梁，让龙虾能控制我的浏览器。

转折：发现这不仅是「登录工具」，更是「给龙虾装上眼睛」。

成果：解决了公众号/小红书/知乎/即刻等所有登录墙问题。

**js-comfyui-client（最实用的技能）**

起源：ComfyUI 工作流太复杂，参数调整靠记忆，每次都要翻笔记。

解决：把常用工作流封装成 CLI，一条命令生成图片/视频。

特色：支持 Z-Image Turbo（9步快生）、Wan 2.2 视频生成。

数据：养虾日记 KL11/12 的配图全部用它生成，Neo-Brutalist 风格可控。

**js-x-monitor（最新的技能）**

起源：想追踪 Andrej Karpathy、a16z、OpenClaw 官方动态，手动刷 X 太浪费时间。

解决：定时监控 + 智能摘要 + 飞书通知。

实战：2026 年 4 月 2 日凌晨 2:19，成功抓到 a16z 关于「AI 改变软件开发」的推文。

监控列表：@karpathy、@openclaw、@a16z、@moltbook、@steipete。

### 开发心得

**从「给自己用」到「给别人用」**

第一个版本：硬编码路径、写死配置、只在我电脑上能跑。

技能化改造：提取配置、补充文档、设计 CLI 接口、写 SKILL.md。

意外收获：强迫自己把「潜规则」变成「显式规则」，代码质量反而提升。

**一个技能 = 一个具体问题**

不搞大而全，只做一件事但做到极致。

- js-eyes = 浏览器控制，不涉及数据处理
- js-x-monitor = 监控+通知，不涉及内容分析
- js-comfyui = 图片生成，不涉及提示词工程

**上架不是终点，是起点**

- 用户反馈会暴露你没想到的场景
- 版本迭代让技能越来越 robust
- 社区贡献让你获得意外收获（比如被官方转发）
