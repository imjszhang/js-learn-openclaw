## 第一件事：给它一块能记事的脑子

头几天我啥也没折腾。

就烦了一件事：每次打开聊天，都像第一次见面。

我上周说过：我是开发者，别给我科普散文。

这周它又：「首先让我为您解释什么是 Git。」

我差点把窗口关了。

**不是它笨。是我没给它记性。**

OpenClaw 官方有个插件叫 memory-core。

三步搞定。

第一步，在 OpenClaw 的插件列表里找到 memory-core，点启用。

它在 `extensions/` 目录下，属于内置插件，不用额外装。

第二步，进 `~/.openclaw` 的配置，把记忆槽位指向 memory-core。

确认 `memorySearch.enabled` 是开的。

不开的话，`memory_search` 和 `memory_get` 这两个工具不会被创建，等于白装。

第三步，正常聊天。

memory-core 会在会话预热、文件变更、搜索触发的时候自动同步索引。

你不用手动"存"什么，它自己攒。

有一点要注意。

memory-core 的「保存」不是你想的那种"把聊天记录存进数据库"。

说人话就是：它本质是索引同步。

把你工作区里的文件、笔记建好索引，下次对话时能快速找到相关内容。

**质量取决于你喂进去的东西，而不是聊了多少句。**

有一天我在飞书里随口扔了句挺刁的：本地部署和云端，到底谁更安全？

它回来的口气，像读过我的日记。

不是泛泛而谈。

是带着「我知道你是谁」的那种回答。

那一下我有体感了。

不是「哇模型好聪明」。

是**它真的记住我了。**

## 第二件事：给它一双眼睛

大概第四天开始，我又烦了另一件事。

你让龙虾帮你看一篇公众号文章，它说「抱歉，我无法访问该链接」。

你甩一个知乎回答，它说「我看不到内容」。

**你跟它之间隔了一面登录墙。**

你看得见的东西，它全看不见。

现在装好龙虾，飞书微信都是通的。

能说话不稀奇了。

看不见，才是真的瞎。

我装的是 js-eyes。

一个浏览器插件。

装完，右键点一下，整篇文章——包括登录墙后面的——刷地送进龙虾的嘴里。

第一次用，我把一篇微信公众号长文甩过去。

它消化了，回了一段摘要。

那一下特别实在。

**不是教程截图底下那种绿色对勾，是龙虾真的「看到」了你看到的东西。**

**先给它一双眼睛。**

能看见，才能开始帮你干活。

## 第三件事：给它搭一座图书馆

有了记性，有了眼睛。

龙虾终于能记住你、能看见外面了。

但你很快会发现一个新问题。

**它的记忆全是碎片。**

你今天扔一篇公众号，明天甩一个知乎回答，后天丢一段 GitHub README。

龙虾记是记了，但东西散得到处都是。

你问它一个跨领域的问题，它翻半天翻不到。

**缺的不是记性。是一座能分类、能检索、能长出结构的图书馆。**

我搭了两套东西。

---

**第一套：收集器。**

开源项目叫 js-knowledge-collector。

干的事很简单。

你在聊天里扔一个链接，它去抓全文、用 AI 生成摘要和标签、存进本地 SQLite 数据库。

想同步到 Flomo 也行，配一下就自动推。

十几个平台都能抓。

微信公众号、知乎、即刻、小红书、GitHub、Reddit、B 站、YouTube。

基本上你平时看的都覆盖了。

公开页面它直连抓。

遇到登录墙的——即刻、小红书这类——它调第二件事里装的 js-eyes，复用你浏览器的登录态，照样抓。

B 站和 YouTube 有专用的抓取器，连字幕都能拿下来。

有个关键设计。

你扔链接的时候，它只往 inbox 排队，不会卡你当前对话。

后台有个定时任务，一键配好。

隔离会话批量处理。

抓取、总结、打标签、入库，全自动。

你白天随手扔，晚上它自己消化完。

装法也不难。

clone 仓库，在 OpenClaw 配置里加载插件路径，跑一次 setup 命令。

登录型平台需要先装好 js-eyes，让收集器能借浏览器登录态。

---

**第二套：棱镜。**

开源项目叫 js-knowledge-prism。

如果说收集器是感官，棱镜就是大脑。

收集器把东西搬进来了，但全是散乱的素材。

棱镜做的事是：把你积攒的笔记和素材，拆成最小的「原子观点」。

然后把相关的原子归成「观点组」。

最后从观点组里蒸馏出更高层的结论。

三层蒸馏。

journal（原始素材）→ pyramid（拆解结构）→ outputs（成品）。

灵感来自《金字塔原理》那本书，但全程是 AI 在跑。

你不用手动分类。

跑一次 `openclaw prism process`，新笔记就被消化进金字塔了。

它只处理增量。

已经拆过的不会重复碰，新加的自动接上。

配好定时任务以后，棱镜自己转，你甚至不用管它。

装法：clone 仓库，在 `openclaw.json` 的 `plugins.load.paths` 里指向棱镜的插件目录，配好 AI API 环境变量。

之后在对话里就能用 `knowledge_prism_process` 等工具，或者终端跑 `openclaw prism` 命令。

---

**最后一步：把图书馆接进记忆。**

收集器和棱镜各自会产出一堆文件。

但龙虾的 memory-core 不会自动去翻它们。

你得告诉它去哪找。

在 `~/.openclaw` 的配置里，把 `memorySearch.extraPaths` 指向两个目录。

收集器的摘要导出目录，和棱镜的高价值层导出目录。

棱镜不是什么都往记忆里塞。

它挑食。

只导出观点组、核心结论、视角背景这些高密度的东西。

原始日记、原子表格、长文成品不进记忆，太吵。

三件东西接上以后。

**收集器管进货，棱镜管加工，memory-core 管上架。**

有一天我问龙虾一个挺杂的问题。

它把三个月前一篇公众号的观点、我自己笔记里的一段判断、还有一个 GitHub README 里的细节，串在一起回了我。

**那一下我知道：这不是聊天了。这是一个有图书馆的龙虾。**

## 没有标准顺序这回事

读到这你肯定想问：必须 1-2-3 吗？

不用。

有人先装眼睛再搭知识库。

有人先搞知识库再喂记忆。

也有人三件同时推。

**都行。**

三件事像三根绳。

你攥哪根先使劲，看你此刻最缺啥。

**你的龙虾，你的顺序。**
