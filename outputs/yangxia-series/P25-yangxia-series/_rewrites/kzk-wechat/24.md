凌晨三点，我的两只“龙虾”还在干活。

隔壁用 Claude Code 的哥们，对着屏幕骂了一宿。

为什么？

AMD 的 AI 负责人甩出 23 万次调用记录，实锤了：Claude Code 变蠢了，甚至开始摆烂。

复杂工程？根本干不了。

评论区炸了。

有人说"Anthropic 开始收割韭菜了”。

有人说"AI 泡沫要破了”。

还有人说“模型退化是商业阴谋”。

但我想说：**你们都骂错对象了。**

Claude Code 变蠢，根子不在模型。

而在 Harness。

不是 Claude 不行了，是 Claude Code 这个产品的设计哲学，从一开始就错了。

今天我就借这把火，聊聊为什么**现在的所谓 Agent，本质上都是垃圾**。

以及 OpenClaw 是怎么把这坨屎变成金子的。

---

先澄清一个概念。

Claude Code 变蠢，不等于 Claude 变蠢。

模型还是那个模型。

但产品体验，烂透了。

为什么？

因为 **Claude Code 是一个“死”的框架**。

它的命脉捏在谁手里？

Anthropic 给的软件框架 + Claude 大模型本身。

作为使用者，你能做什么？

**什么都做不了。**

你只能等。

等 Anthropic 更新软件。

祈祷 Claude 模型别退化。

你用了三个月，它的代码补全更懂你的风格了吗？

没有。

它遇到你项目的特定坑，会自己学习吗？

不会。

它出厂时是个智障，现在还是个智障。

你的使用，不会让它变得更好用。

这就是“死框架”的本质：**能力是开发者预设的，不是用户用出来的。**

所以模型一退化，整个产品直接崩盘。

因为除了模型，它没有任何护城河。

---

再看 OpenClaw。

它也依赖大模型。

但它多了一个东西：**Harness 工程**。

Harness 是什么？

说人话就是：你驾驭 AI 的那套系统。

系统指令、工具配置、记忆管理、工作流设计。

在 OpenClaw 里，这套东西不是 Anthropic 或 OpenAI 给你定死的。

而是**你自己可以通过对话持续迭代的**。

关键区别来了：

| | Claude Code | OpenClaw |
|--|-------------|----------|
| **框架性质** | 死的，开发者预设 | 活的，用户可迭代 |
| **使用者能做什么** | 只能用，不能改 | 通过对话持续优化 Harness |
| **时间效应** | 越用越失望 | 越用越强 |
| **对大模型的依赖** | 100%（唯一变量） | 逐渐降低（Harness 兜底）

这就是我说的“垃圾”与“非垃圾”的区别。

Claude Code 这类产品，把用户当“使用者”。

你只管输入，别的别碰。

OpenClaw 把用户当“驯养师”。

你可以通过持续对话，让你的“龙虾”越来越懂你的需求。

**你是在养它，不是在用它。**

---

我用 OpenClaw 四个月。

对话了 800+ 次。

现在我的养虾系统长什么样？

- **4 个子 Agent**：情报组、资料组、知识组、汇总组，并行干活。
- **独立指令**：每个 Agent 都有自己的系统指令和工具链。
- **实时微调**：每次对话都在微调指令，贴合我的工作流。
- **模型无关**：就算换个底层模型，我的 Harness 依然有效。

这叫什么？

这叫 **Harness 的自我强化循环**。

每次我用 OpenClaw 完成一个任务，我都会反思。

哪里卡住了？

哪里可以自动化？

然后我把这些洞察，直接写进 Harness。

系统指令、记忆规则、工具配置。

结果是：**我对大模型性能的依赖越来越低。**

因为好的 Harness 可以弥补模型的不足。

模型变蠢了？

没关系。

我的系统指令够清晰，记忆够完整，工作流够顺畅。

整体体验不会崩盘。

而 Claude Code 用户呢？

他们只能干瞪眼。

等 Anthropic 修 bug。

---

这让我想到一个更深层的结论。

传统 AI 工具（包括 Claude Code），本质上是**开发导向**的。

开发者定义功能，用户只管使用。

OpenClaw 是**应用生长导向**的。

你通过持续对话，让系统在使用中自我演化。

这就是编程范式的第三次革命：

**命令式编程**：告诉计算机每一步怎么做。

**声明式编程**：告诉计算机想要什么结果。

**对话式编程**：通过对话让系统自我演化。

前两种范式，人类都是“设计者”。

对话式范式，人类是“对话者”。

你不设计系统。

你和系统对话。

让它自己长出来。

这不是降级。

是升级。

因为**系统的复杂度已经超过了人类的预见能力**。

你无法在设计阶段就考虑所有边界情况。

唯一的办法，是让系统在使用中自我适应。

---

现在再看那个新闻。

Claude Code 变蠢，是因为它的 Harness 是死的。

它没有给用户提供任何自我优化的空间。

所以一旦模型性能波动，整个产品就暴露无遗。

而 OpenClaw 用户没有这种焦虑。

因为我们知道：**Agent 的上限不是模型决定的，是 Harness 决定的。**

模型只是引擎。

Harness 才是整车。

引擎再强，装在拖拉机上也是拖拉机。

引擎一般，但 Harness 设计得好，照样能跑赢。

这就是为什么我说，从 Harness 角度来看，现在的所有 Agent 都是垃圾。

**不是它们现在不强，是它们不会生长。**

而生长，才是 Agent 和工具的本质区别。
