## 微信把你的数据锁了起来

先说原理，很简单。

微信在 Mac 上把所有聊天记录和收藏，都塞进了一个加密数据库。

SQLCipher 加密，AES-256-CBC 逐页加密。

听着很吓人对吧？

但有一个漏洞。

密钥不在数据库里。

在你自己电脑的进程内存里。

wechat-cli 做了一件很直接的事。

用一段只有 320 行的 C 代码，扫描微信进程的内存。

找到密钥。

然后逐页透明解密。

**纯本地。只读。零网络。**

数据不离开你的电脑。

不需要 sudo 权限去跑查询。

只有第一次提取密钥需要授权，之后所有操作都不需要。

这意味着什么？

意味着龙虾不需要你转发了。

它可以通过 wechat-cli，直接"走"进你的微信数据仓库，把想要的东西拿出来。

说人话就是：

**它拿到了你家保险柜的钥匙，而且是在你睡觉的时候偷偷配的。**

## 11 条命令，把微信数据全释放出来

wechat-cli 提供了 11 条命令。

我不逐条展开了，挑几个跟知识管理最相关的说。

**收藏夹导出。**
一条命令，把微信收藏夹里的文章、图片、视频、文本全部导出成 Markdown 文件。
自动带上时间戳和来源信息。
干干净净的文本格式——恰好是 Prism 最喜欢的输入。

**聊天记录导出。**
按群聊、按联系人、按时间范围导出。
三个月前技术群里关于架构选型的那场讨论？
一条命令拉出来。

**增量新消息。**
只返回上次检查之后的新内容。
配合定时任务，每天自动追加新的收藏和聊天。

**关键词搜索。**
不用打开微信翻聊天记录，一条命令全局搜索。
让 JS_CLAW 定位你半年前的某条收藏，几秒的事。

**群聊统计。**
哪个群发言最活跃？谁产出价值最高？数据说话。

核心突破是：

之前你给龙虾喂数据的方式是"手动转发"。

现在变成"龙虾自己去取"。

从推送模式，变成拉取模式。

## 收藏夹不用一篇篇转了

回到那个读者的问题。

有了 wechat-cli，收藏夹的处理方式完全变了。

**之前的流程（KL21）：**
看到好文章 → 点收藏 → collector 自动抓取链接 → AI 摘要 → 入库

听起来很自动化对吧？

但你手动收藏的那部分，collector 是抓不到的。

你只能自己转发给龙虾。

累不累？

**现在的流程（KL27）：**
wechat-cli 一键导出收藏夹 → Markdown 文件 → 丢进 Prism 的 journal 目录 → 龙虾自动处理

不需要你操作。

导出后的 Markdown 文件，每条消息一行，带时间戳、发言人、消息类型标注。

JS_CLAW 收到之后，自动走 Prism 的处理流程：

Atoms——从文章标题和摘要中提取信息单元。

Groups——按主题归类。

Synthesis——收敛出顶层观点。

从"你一篇篇转发"到"龙虾自己吃"。

你就说牛不牛逼？

## 但收藏夹只是开胃菜

解决完收藏夹的问题，我发现 wechat-cli 能做更大的事。

微信里真正有价值的，可能不是收藏夹。

是群聊。

产品群里敲定的需求变更。

技术群里架构师的随口建议。

项目群里讨论达成的共识。

这些从来没有被文档化。

但它们是你工作中最有价值的"暗知识"。

wechat-cli 可以把整个群聊的聊天记录导出来，变成结构化的 Markdown 文件。

然后丢给 JS_CLAW。

Prism 会自动做三件事：

第一，提取信息单元。
"老张说 A 方案更好"。"小李提到 B 框架有坑"。"最终决定用轻量方案"。

第二，归类。
"架构选型讨论"归到一起。"技术踩坑记录"归到一起。"产品需求变更"归到一起。

第三，收敛观点。
"团队最终倾向轻量方案"。"缓存方案存在一致性风险"。

你不需要手动整理聊天记录。

龙虾自己读。自己分类。自己总结。

**不是全文搜索，是语义级理解。**

你可以直接问 JS_CLAW："上个月技术群讨论过哪些架构选型？"

它会告诉你答案。

这感觉，就像有个秘书把你过去三年的废话都整理成了决策报告。

## 让知识库每天自动成长

更爽的是，这套东西可以全自动。

wechat-cli 有一个 `new-messages` 命令，维护本地状态文件。

每次只返回上次检查之后的新消息。

配合 cron 定时任务：

每天晚上，自动获取新的收藏和聊天记录。

追加到 journal 目录。

Prism 自动处理。

你的知识库每天自动增长。
