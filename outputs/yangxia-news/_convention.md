## §4 文章结构（七段式）

每篇文章必须遵循以下骨架。段落可合并或微调，但七个功能块不可缺失。

### H. 标题 + 导读

**标题公式**：`主体 +「动作」+ 结果/影响`

- 标题必须包含至少一个「」包裹的关键动作
- 标题必须传递冲突或戏剧性（反转、碾压、封杀、暴涨等）
- 长度 15-30 字

**导读格式**：

```
【养虾快讯】一段话概述（≤3 句，≥3 个关键数字），点明核心事件和最大看点。
```

### ① 爆点开场（1-3 段）

- 用最炸裂的数据、事实或冲突砸开
- 第一句话必须让人"停下来看"
- 必须包含至少 1 个具体数字

### ② 事件主线（3-6 段）

- 按时间线或因果链叙述核心事件
- 每段聚焦一个关键节点
- 可穿插当事人引语（blockquote 格式）
- 传送门链接标注关键仓库/帖子

### ③ 人物/案例特写（2-4 段）

- 给事件一个"人味儿"的锚点
- 可以是事件当事人，也可以是 JS 的实践佐证
- 引用具体言行，不做泛泛描述

### ④ 技术深潜（核心干货）

- **编号列表**，每条一句话标题加粗 + 2-3 句通俗解释
- 数量通常 4-8 条
- 用非程序员能懂的语言解释技术要点
- 禁止贴大段代码，可用极短的命令行示例（≤2 行）

### ⑤ 社区反应 / 延伸发现（2-4 段）

- 业界大佬的点评（附出处）
- 社区的二次创作、衍生项目
- 有趣的隐藏发现

### ⑥ 原因/争议/背景补充（2-3 段）

- 追问"为什么会发生"
- 提供深层技术/商业背景
- 可引入不同观点的碰撞

### ⑦ 收尾金句 + 前瞻（1-2 段）

- 一句话定性事件意义
- 前瞻后续关注点
- 语气可以俏皮/犀利

## §5 风格指纹

### 5.1 语气与用词

| 规则                     | 说明                                                           |
| ------------------------ | -------------------------------------------------------------- |
| **「」用法**             | 技术概念、关键动作、隐喻一律用「」包裹                         |
| **情绪词**               | 允许使用"杀疯""裸奔""炸锅""碾压"等新媒体用语，但不可堆砌     |
| **数据密度**             | 每 2-3 段至少一个关键数字                                      |
| **段落长度**             | 单段不超过 5 句话，大部分段落 1-3 句                           |
| **加粗**                 | 每段的核心观点句加粗，每篇加粗不超过 15 处                     |
| **感叹号**               | 标题和导读可用，正文每篇不超过 5 个                            |

### 5.2 引用与链接

| 格式       | 用法                                                           |
| ---------- | -------------------------------------------------------------- |
| **传送门** | 关键链接用"传送门：URL"标注                                    |
| **引语**   | 当事人原话使用 blockquote 格式                                 |
| **名人背书** | 可引用业界知名人物的公开点评（Karpathy、马斯克等），需注明出处 |
| **参考资料** | 文末统一列出所有引用的链接                                     |

### 5.3 图片占位

- 用 `[图片]` 标记插图位置
- 每 3-5 段至少一个 [图片]
- 关键数据/架构处建议配图

## §6 系列惯例

1. **导读格式**：`【养虾快讯】` + 一段话概述
2. **篇尾格式**：参考资料列表 + `以上～关注养虾快讯，追踪 AI Agent 生态最前沿。`
3. **篇号**：按发布顺序编号（第 1 期、第 2 期……）
4. **与 P25 联动**：如事件与养虾日记某篇高度相关，可在文末注明"延伸阅读：《JS 的养虾日记 · 第 N 篇》"
5. **JS 实践引用**：引用 JS 实践时使用第三人称（"开发者 JS"），不用"我"
6. **时效性标注**：导读末尾标注事件发生日期

---

## §7 硬性禁止

1. **禁止第一人称**：不得使用"我""我们"（引语 blockquote 内除外）
2. **禁止编造数据**：所有数字必须有来源（journal 素材、公开报道或 GitHub 数据）
3. **禁止贴大段代码**：技术要点用通俗语言解释，极短命令行示例（≤2 行）可保留
4. **禁止软文腔**：不得替任何产品/公司做无依据的正面背书
5. **禁止无来源判断**：观点必须有数据或引语支撑，不得凭空下结论
6. **禁止与 P25 已发布内容矛盾**：时间线、实体关系、关键数据必须与养虾日记一致
7. **禁止脱离事件的泛泛而谈**：每篇必须锚定一个具体事件，不得写成"AI 行业总览"
8. **禁止标题党无实质内容**：标题中的每个吸引点，正文中必须有对应的实质内容
